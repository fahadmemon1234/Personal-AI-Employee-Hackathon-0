# Agent Skill: Cross-Domain Integration

## Skill ID: `cross_domain_integrate`

**Version:** 1.0  
**Created:** 2026-02-23  
**Author:** AI Digital FTE

---

## Purpose

Automatically scan `/Needs_Action` for pending items from the last 24 hours, classify them as Personal or Business (or both), create integrated Plans in `/Plans`, link cross-domain workflows, and summarize status in `Dashboard.md`.

---

## Trigger Conditions

- Manual invocation by user
- Scheduled daily execution (recommended: every morning)
- After batch import of new items into `/Needs_Action`

---

## Input

- `/Needs_Action/` directory contents
- Current date/time for 24h window calculation
- `Company_Handbook.md` rules
- `Dashboard.md` for status updates

---

## Output

- Plan files in `/Plans/` with naming convention: `Plan_CrossDomain_[timestamp]_[domain].md`
- Updated `Dashboard.md` with "## Cross-Domain Status" section
- Summary output: "Plans created. Ready for next phase." or error details

---

## Procedure

### Step 1: Scan /Needs_Action (Last 24h)

1. List all files in `/Needs_Action/`
2. Filter by file modification date or filename timestamp (last 24 hours)
3. Read each file to extract:
   - Source type (email, whatsapp, notification, file)
   - Sender/Contact
   - Content/Subject
   - Keywords (urgent, payment, invoice, help, etc.)
   - Timestamp

### Step 2: Classify Each Item

| Domain | Criteria | Examples |
|--------|----------|----------|
| **Personal** | Individual communications, job applications, personal banking, learning | Gmail notifications, LinkedIn job alerts, Bank Alfalah statements, WhatsApp personal chats |
| **Business** | Client communications, invoices, payments >$100, Odoo/ERP, social media | Odoo messages, client invoices, payment approvals, business WhatsApp groups |
| **Cross-Domain** | Items spanning both domains | WhatsApp client request → invoice → Odoo entry → email confirmation |

**Classification Keywords:**

- **Personal:** "job alert", "application", "bank statement", "LinkedIn", "connection accepted", "learning"
- **Business:** "invoice", "payment", "Odoo", "client", "demo", "project", "social media", ">$100"
- **Urgent:** "urgent", "ASAP", "deadline", "final call", "last chance"

### Step 3: Create Plans

For each classified item (or grouped cross-domain items), create a Plan file:

**Naming Convention:**

### Step 4: Link Cross-Domain Workflows

For items that span multiple domains:

1. Identify the workflow chain (e.g., WhatsApp → Invoice → Odoo → Email)
2. Create a **master plan** that references individual step plans
3. Add cross-references in each plan's "Cross-Domain Links" section
4. Ensure approval gates are placed before irreversible actions

**Example Cross-Domain Flow:**
```
WhatsApp Client Request (Business)
         ↓
Create Invoice (Business + Financial)
         ↓
Odoo Entry (Business System)
         ↓
Email Confirmation (Business Communication)
         ↓
Bank Alert Verification (Personal Banking)
```

## Rules Compliance (Company_Handbook.md)

| Rule | Implementation |
|------|----------------|
| **Efficiency First** | Automated scanning and classification |
| **Data Integrity** | Accurate timestamps, file references, audit trail |
| **Security** | No auto-execution of payments or irreversible actions |
| **Transparency** | All actions logged in plans and dashboard |
| **Proactivity** | Flags urgent items and high-value payments |
| **Polite Tone** | All generated communications maintain professionalism |
| **Payment Flag** | Any payment >$100 requires human approval |
| **Urgent Priority** | Items with urgent keywords are prioritized |

---

## Error Handling

| Error | Action |
|-------|--------|
| Cannot read /Needs_Action | Log error, output error message, exit gracefully |
| No items in 24h window | Output "No new items in last 24h. Ready for next phase." |
| Plan creation fails | Log specific error, continue with remaining items |
| Dashboard update fails | Create backup file, log error |
